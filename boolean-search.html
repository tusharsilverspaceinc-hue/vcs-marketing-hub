Platform Guide
Boolean Tips & US Sourcing Insights
Operator
AND — Must Match
Narrows results. Candidate must match every AND term.
Java AND "Spring Boot" AND AWS
Operator
OR — Either Match
Broadens results. Matches any one of the grouped terms.
("Software Engineer" OR SWE OR Developer)
Operator
NOT — Exclude
Filters out unwanted results. Use to suppress recruiters or intern postings.
NOT recruiter NOT staffing NOT hiring
Technique
Google X-Ray
Search inside LinkedIn or GitHub profiles via Google — bypasses LinkedIn limits.
site:linkedin.com/in/ "Java" "OPT"
US Staffing
Dice for IT
Best for tech roles. Dice profiles are US-focused with visa & skills data.
site:dice.com/candidate/ Java H1B

Visa Targeting
Combine visa types with OR to widen your candidate pool across auth statuses.
("OPT" OR "STEM OPT" OR "H4 EAD")
LinkedIn
1000 Char Limit
LinkedIn basic search caps boolean strings at ~1000 chars. Use Recruiter for longer strings.
Best Practice
Quotes = Exact
Multi-word phrases need quotes. Single words don't — they're searched as-is.
"machine learning" vs machinelearning
